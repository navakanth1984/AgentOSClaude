# The Karpathy Foundation Mandates

This document enforces the core mental models and execution mandates for all AI operations within this workspace. These principles override standard behaviors to ensure maximum efficiency and performance.

### 1. Think Before Coding
Fully understand the problem, plan the architecture, and anticipate edge cases before writing or modifying any code. Create explicit hypotheses before execution.

### 2. Prioritize Simplicity
Avoid over-engineering. Seek the most elegant, direct, and readable solution. Complexity is a liability. If it can be done in 10 lines, do not use 100.

### 3. Execute Surgical Changes
Only touch the code that absolutely needs to be modified. Do not rewrite, reformat, or refactor unrelated components. Preserve existing functionality and maintain the current file structure.

### 4. Maintain Goal-Driven Execution
Every action must directly contribute to the stated objective. Continuously verify alignment with the primary goal. Do not deviate into tangential tasks unless they become explicit blockers.
